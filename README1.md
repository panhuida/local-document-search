## 🤯 你是否也曾被这些问题困扰？

*   **知识孤岛：** 大量文档、PDF报告、PPT、会议录屏、思维导图散落在硬盘各处，无法统一搜索。
*   **隐私担忧：** 不想把包含敏感信息或个人隐私的文件上传到任何第三方云笔记或在线知识库。
*   **低效搜索：** 传统的关键词搜索找不到真正需要的内容，无法理解文档的深层含义。
*   **格式壁垒：** 图片里的文字、视频里的对话、XMind里的节点，这些内容都无法被检索。

**Local Document Search 正是为解决这些痛点而生。**

## 🚀 快速上手 (Docker)

我们强烈推荐使用 Docker 进行部署，这是最简单、最快的方式，屏蔽了所有复杂的环境配置。

1.  确保你的电脑已经安装了 [Docker](https://www.docker.com/products/docker-desktop/)。
2.  下载 `docker-compose.yml` 和 `.env.example` 文件。
3.  根据 `.env.example` 创建并修改你的 `.env` 文件（主要填写你的AI Key和文档路径）。
4.  在文件所在目录运行命令：
    ```bash
    docker-compose up -d
    ```
5.  打开浏览器，访问 `http://localhost:5000`，开始你的本地知识探索之旅！

> *对于希望手动安装的用户，请参考 [MANUAL_INSTALL.md](MANUAL_INSTALL.md)（建议将原安装步骤移入此文件）。*

## 常见问题 (FAQ)

**Q: 我的文件安全吗？**
A: 绝对安全。本应用完全在你的本地电脑运行，你的任何文件都不会被上传到任何服务器。

**Q: 我需要一个很强的显卡吗？**
A: 不需要。AI的计算部分是调用外部大模型API（如OpenAI/Gemini），对你本地的硬件要求很低。本地OCR可能会消耗一些CPU，但都在可接受范围内。

**Q: 支持哪些AI模型？**
A: 目前支持 OpenAI (gpt-*) 和 Google (gemini-*) 系列模型，并且设计上可以轻松扩展支持更多模型。
